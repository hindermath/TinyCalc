# Workflow Notes

## Pull Request Handling

- Pull requests are reviewed, merged, and then closed by the repository owner.
- For follow-up work, always create a new `codex/*` branch and a new pull request.
- Each pull request should include a dedicated PR description markdown file under `docs/`.

## Lastenheft Queue

- The visible processing order for prepared requirements documents is recorded in
  `docs/Lastenheft_Abarbeitungsreihenfolge.md`.
- Treat that file as preparation for later Spec Kit runs. It does not start a
  branch, specification, plan, task list, test change, or implementation by
  itself.
